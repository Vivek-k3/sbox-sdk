title: Anonymous usage data
description: sbox SDK collects limited anonymous usage data to understand adoption and improve the SDK.

## What we collect

sbox SDK collects limited anonymous usage data by default. We use it to understand adoption, see which parts of the SDK are used, prioritize improvements, and trend sandbox performance across providers.

The data is intentionally coarse and unlinkable to a person or tenant:

- SDK version
- runtime and platform
- selected provider
- operation type, such as creating a sandbox or running a command
- success or failure
- duration bucket and raw duration in milliseconds
- error category
- command exit code
- create vs connect lifecycle (`create` | `connect`)
- non-secret create config dims when supplied: template / image / blueprint id, region, vcpus, memory_mb, disk_mb, has_gpu, ttl_ms, on_idle, port_count, and presence flags for env / volumes / secrets

`template` is the caller-supplied id (image, blueprint, snapshot, or runtime). It may reflect private naming in your org; opt out if that is sensitive.

Usage data may be sent from any environment where the SDK runs, including local development, CI, tests, previews, and production.

This is separate from any analytics you use in your own app. sbox SDK does not read, modify, identify, or share state with your app's analytics client.

## What we never collect

sbox SDK does not collect:

- API keys or credentials
- prompts or model inputs
- command strings
- file paths or file contents
- environment variable keys or values
- secret keys or values
- stdout or stderr
- sandbox ids
- project or sandbox names
- metadata values
- GPU model strings
- provider-specific request payloads

## How to opt out

Set any of these environment variables:

```bash
SBOX_TELEMETRY_DISABLED=1
SBOX_DISABLE_TELEMETRY=1
SBOX_TELEMETRY=0
DO_NOT_TRACK=1
```

Or disable usage data for one client:

```ts
import { createSandboxClient } from "sbox-sdk";

const client = createSandboxClient({
  telemetry: false,
});
```

Usage data is not automatically disabled in CI, tests, local development, previews, or production. Use one of the opt-out controls above when you do not want it sent.
